## 4. 核心需求

- 站点采用偏开发者路线的个人主页结构。
- 一级导航包含 `Home`、`About`、`Projects`、`Writing`、`Contact`。
- 首页突出自我定位、当前状态、主项目入口和最新笔记入口。
- 站点可长期迭代，内容以本地文件和组件化方式维护。

## 5. 方案概述

- 技术选型：`Next.js + TypeScript + Tailwind CSS + MDX + Framer Motion`
- 目录结构：按页面、组件、内容、配置分层组织。
- 页面/模块划分：`Home` / `About` / `Projects` / `Writing` / `Contact`
- 内容管理方式：先本地文件管理，后续视需要再引入 CMS。

## 7. 交互与体验原则

- 首屏目标：快速说明“你是谁、做什么、现在关注什么”。
- 导航策略：结构简单，一级导航固定且清晰。
- 动效策略：只在首屏、卡片 hover、页面切换等关键位置使用。
- 移动端策略：优先保证可读性和易操作性，不追求复杂交互。

## 8. 数据与内容

- 内容来源：本地 Markdown / MDX 文件、配置文件、静态资源。
- 是否需要动态数据：首版不需要。
- 是否需要 CMS：首版不需要，后续内容量增加后再评估。

## 9. 非功能需求

- 性能：首屏加载轻量，图片和动效需控制体积。
- SEO：基础元信息、语义化结构、可抓取内容必须具备。
- 可维护性：内容与组件分离，目录清晰，方便长期迭代。
- 可扩展性：后续可平滑增加博客标签、搜索、订阅、CMS。

## 10. 风险与约束

- 风险：初期项目数量少，首页内容可能偏空。
- 约束：不做实验室页，不引入重后台。
- 未决问题：项目页首批展示内容的具体来源。

## 11. 验收标准

- 一级导航结构清晰且完整。
- 首页、关于、项目、写作、联系五个页面职责明确。
- 个人站本身可作为第一个项目展示。
- 首版不依赖 CMS 也能完成内容更新。
