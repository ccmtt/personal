# 开工清单

> 用于每次重新进入项目时快速恢复上下文，避免直接开写导致目标漂移。

## 1. 先读这些文件

按顺序快速浏览：

1. `GUIDE.md`
2. `ARCHITECTURE.md`
3. `HLD.md`
4. `SDD.md`
5. `TEST.md`
6. `README.md`
7. `GIT_WORKFLOW.md`

## 2. 先确认当前状态

- 看 `git status`
- 看当前分支
- 看远程仓库是否已配置
- 看目录里是否已经有前端项目骨架

## 3. 先明确本次目标

本项目第一版目标：

- 搭起个人站基础结构
- 先完成 `Home / About / Projects / Writing / Contact`
- 不上 CMS
- 不做实验室页
- 先用本地内容文件和组件实现

## 4. 第一次开工顺序

### 如果还没有前端项目

1. 初始化 Next.js 项目
2. 安装依赖
3. 配置 Tailwind CSS
4. 配置基础目录结构
5. 建立路由页面
6. 放入基础布局组件
7. 运行本地开发服务器

### 如果已经有前端项目

1. 检查依赖是否完整
2. 启动本地开发服务器
3. 对照 HLD / SDD 补齐页面结构
4. 先做首页

## 5. 本次优先级

建议按这个顺序推进：

1. `Home`
2. `Layout / Navigation / Footer`
3. `About`
4. `Projects`
5. `Writing`
6. `Contact`

## 6. 每次开发前的检查

- 设计文档是否足够清楚
- 本次任务是否足够小
- 是否需要先补测试再写代码
- 是否有明确的提交点

## 7. 每次收工前的检查

- 当前变更已提交
- 提交信息可读
- 已推送到远程仓库
- `git status` 没有遗漏关键文件
- 下一步待办已记录

## 8. Claude Code 使用方式

- 先让它读 `GUIDE.md` 和 `ARCHITECTURE.md`
- 再让它按 `HLD.md` 和 `SDD.md` 执行
- 如果要写代码，先确认本次实现范围
- 每完成一个小步骤就提交一次
